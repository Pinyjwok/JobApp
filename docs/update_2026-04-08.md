## What We've Been Doing

We've been running the CV optimization pipeline end-to-end using a test candidate persona (Chloe Simmons, Early Childhood Educator applying to City of Melbourne). Over 6 test cycles on this persona, we've systematically found and fixed bugs across all 17 agents in the pipeline.

---

## Where We're At

The pipeline now completes end-to-end for the first time — from raw CV upload through to a tailored CV and cover letter output. This is a meaningful milestone; earlier runs were stalling mid-pipeline.

**What's working well:**
- Company research, gap analysis, and quality review are solid
- The integrity check correctly catches fabricated claims before they reach the final CV
- Style and tone analysis are functioning
- The assembly phase (all 8 CV-building steps) runs through to completion

**What was still broken in TC06 (now fixed):**
- The integrity failure handler was silently ignoring flagged claims and marking the CV as complete anyway — fabrications could reach the output. This was the most serious issue and is now resolved.
- Several agents were writing data in the wrong format, meaning downstream agents couldn't read each other's output reliably. All schema mismatches have been corrected.
- One agent (Credentials Formatter) was occasionally skipping entirely without explanation. A startup validation guard has been added.
- The pipeline tool limit was causing the Tone Analyst to abort before saving state, stalling the run. Fixed by reducing redundant file reads.

## Skills / Tooling Note

Testing uses a custom `/test-agent` skill built into the Claude Code workspace. After each agent completes in KEMU, the tester pastes the output into Claude Code and receives an instant bug report with severity ratings and a running log. This has dramatically reduced the time to identify and fix issues compared to manual review.

For running new persona tests, a Seed B snapshot exists (`/Users/piny/JobApp/docs/snapshots/pre_assembly/chloe_simmons/`) — a pre-built state file that skips the main pipeline and drops straight into CV assembly. This lets us test the assembly phase in isolation without re-running research and analysis each time, saving significant time per run.

## Recommendation: New Test Persona for TC07

Six runs on the same persona has given us good coverage of the entry-level/care-sector pattern. The pipeline now needs to be tested against a different candidate profile to surface failure modes that Chloe's profile doesn't trigger.

**Suggested approach for the next persona:**

1. Choose a profile type that contrasts with Chloe — a career-changer or senior professional would stress different parts of the pipeline (gap analysis, history formatting with many roles, profile building with a split narrative).
2. Create the persona files — `cv_raw.txt` (raw CV text) and `jd_raw.txt` (target job description). Keep the CV realistic but deliberately include some gaps relative to the JD so the integrity check and gap interview have something to work with.
3. Run the full pipeline from Main Orchestrator through to Integrity Check, using the `/test-agent` skill after each step.
4. After a clean run, snapshot the post-Tone-Analyst state as a new Seed B for that persona, so future assembly-only reruns don't need to redo the research phase.
5. Target: zero P0 bugs on TC07. If the pipeline passes with a new persona, it's a strong signal the core logic is stable enough to move toward a wider test group.
