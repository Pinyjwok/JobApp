# CV Optimization System — Update (2026-04-03)

---

## Claude Code: Testing Infrastructure

**Two new skills added:**

- `/test-agent` — run after each KEMU agent completes. Paste in the chat output and Claude Code checks it against the files on disk, the instructions, and what the previous state should've been. Bugs get logged to `tc_running_log.md` with IDs and a quick summary.
- `/test-finalize` — wraps up a test run. Produces a Developer Brief with all bugs categorised by severity (P0–P3), gives the run a verdict/score, and adds it to the history.

**Snapshots added:**
Claude now saves the memory files at two points during a run:
1. **After JD Enhancer** — before the heavy lifting starts (Analyst + Reviewer).
2. **After Tone Analyst** — clean baseline before assembly kicks off.

This means if something goes wrong mid-run, there's a known-good state to reference.
